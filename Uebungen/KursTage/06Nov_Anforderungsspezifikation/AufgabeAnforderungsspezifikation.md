## Background:

Die Stadtverwaltung möchte ein digitales System einführen, das Autofahrern hilft, freie Parkplätze in Echtzeit zu finden. Das System soll über eine App und eine Webplattform verfügbar sein. Sensoren auf den Parkplätzen melden, ob ein Platz frei oder belegt ist. Nutzer sollen Parkplätze suchen, reservieren und bezahlen können.

## Ihre Aufgabe:

1. Formulieren Sie eine Einleitung, in der Sie den Zweck und die Ziele des Smart-Parking-Systems beschreiben.
  # Das Smart-Parking-System soll Autofahrer helfen, schneller einen freien Parkplatz zu finden. Sensoren auf den Parkplatz      erkennen, ob ein Platz frei oder belegt ist. Diese Informationen werden in Echtzeit in einer App und auf einer Website angezeigt.
  # Ziel: weniger Parksuchverkehr, weniger Abgase und mehr Konfort für die Nutzer.

2. Definieren Sie funktionale Anforderungen, z. B. wie Parkplätze angezeigt, reserviert und bezahlt werden.
   # Parkplatzanzeige            - Freie und belegte Parkplätze werden auf einer Karte in Echtzeit angezeigt
   # Suche                       - Nutzer können Parkplätzen in der Nähe oder an einem bestimmten Ort suchen
   # Reservierung                - Ein Parkplatz kann kurzzeitig reseviert werden
   # Bezahlung                   - Parkgebühren können digital bezahlt werden z. B. per APP oder Online-Zahlung
   # Benachrichtigung            - Nutzer bekommen Hinweise über Reservierungen oder wenn ein Parkplatz bald frei wird

3. Definieren Sie nicht-funktionale Anforderungen, z. B. in Bezug auf Sicherheit, Performance oder Verfügbarkeit.
   # Sicherheit                 - Nutzerdaten und Zahlungen müssen geschützt und verschlüsselt sein (Datenschutz!!!)
   # Schnelligkeit              - Die APP soll schnell reagieren
   # Verfügbarkeit              - Das System soll fast immer erreichbar sein (99,99 %)
   # Benutzerfreundlichkeit     - Die APP soll einfach zu bedienen und übersichtlich sein
   # Erweiterbarkeit            - Das System soll später leicht auf andere Stadtbereiche ausgedehnt werden können

4. Erstellen Sie einen kurzen Anhang, z. B. mit einem Glossar und einem Hinweis auf ein mögliches Use-Case-Diagramm.
   # Sensor                     - Gerät das erkennt ob ein Parkplatz frei ist
   # APP / Webplattfort         - Digitale Anwendungen über die Nutzer Parkplätze finden und bezahlen
