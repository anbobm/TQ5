Aufgabe 1

- Erstelle einen neuen Ordner namens "git-uebung".
- Öffne ihn im Terminal.
- Führe "git init" aus.
- Erkläre, was der .git-Ordner bedeutet.
- Erstelle eine Datei "readme.txt" mit etwas Text.
- Führe nacheinander aus: git status, git add readme.txt, git commit -m "Erster Commit".
- Überlege, was "add" und "commit" jeweils machen.
- Ändere den Inhalt von readme.txt.
- Führe "git diff" aus, um die Änderungen zu sehen.
- Mache einen neuen Commit.


Aufgabe 

- Benutze "git log" um den Verlauf der Commits zu sehen
- Führe "git log --oneline" aus.
- Erkläre den Unterschied.


Aufgabe 2

- Erstelle einen neuen Branch mit "git branch feature-text" und wechsle mit "git checkout feature-text".
- Ändere readme.txt und committe.
- Wechsle zurück zu main und vergleiche die Unterschiede.
- Führe "git merge feature-text" aus.


Aufgabe 3

- Lege auf GitHub ein leeres Repository an.
- Füge es mit "git remote add origin <URL>" hinzu.
- Pushe mit "git push -u origin main".
- Füge auf GitHub direkt eine Datei hinzu.
- Hole sie mit "git pull".
- Ändere lokal etwas und pushe wieder mit "git push".
