## Background:

Die Stadtverwaltung möchte ein digitales System einführen, das Autofahrern hilft, freie Parkplätze in Echtzeit zu finden. Das System soll über eine App und eine Webplattform verfügbar sein. Sensoren auf den Parkplätzen melden, ob ein Platz frei oder belegt ist. Nutzer sollen Parkplätze suchen, reservieren und bezahlen können.

## Ihre Aufgabe:

1. Formulieren Sie eine Einleitung, in der Sie den Zweck und die Ziele des Smart-Parking-Systems beschreiben.

2. Definieren Sie funktionale Anforderungen, z. B. wie Parkplätze angezeigt, reserviert und bezahlt werden.

3. Definieren Sie nicht-funktionale Anforderungen, z. B. in Bezug auf Sicherheit, Performance oder Verfügbarkeit.

4. Erstellen Sie einen kurzen Anhang, z. B. mit einem Glossar und einem Hinweis auf ein mögliches Use-Case-Diagramm.

# 1. Einleitung
# Das Smart-Parking-System hilft Autofahrern, freie Parkplätze in Echtzeit zu finden.
# Es soll über App und Web nutzbar sein.
# Ziele:
#  Zeit sparen bei der Parkplatzsuche
#  Verkehr und Abgase reduzieren
#  Einfaches Reservieren und Bezahlen von Parkplätzen

# 2. Funktionale Anforderungen
#  Das System zeigt freie und belegte Parkplätze auf einer Karte.
#  Nutzer können Parkplätze suchen und nach Preis oder Entfernung filtern.
#  Nutzer können Parkplätze reservieren.
#  Nutzer können über App oder Web bezahlen (z. B. mit Karte oder PayPal).
#  Nutzer können sich registrieren und ihr Profil verwalten.
#  Die Stadtverwaltung kann Parkdaten und Statistiken einsehen.

# 3. Nicht-funktionale Anforderungen
#  Sicherheit: Daten sind verschlüsselt (HTTPS, Datenschutz nach DSGVO).
#  Performance: Karte soll schnell laden .
#  Verfügbarkeit: System soll fast immer erreichbar sein .
#  Benutzerfreundlichkeit: einfache, verständliche Bedienung.
#  Skalierbarkeit: System kann für  Städte erweitert werden.

# 4. Anhang
# Glossar:
#  Sensor: erkennt, ob ein Parkplatz frei oder belegt ist.
#  Reservierung: Vorab-Buchung eines Parkplatzes für kurze Zeit.
#  Dashboard: Verwaltungsansicht für die Stadt.
